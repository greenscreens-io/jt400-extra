JT400 Extra Pack for IBM i

A module on top of JT400 improving usage of JT400 ProgramCall and ServiceProgramCall
by helping to map IBM i program parameters and response formats.
Download Blog

Quick introduction

JT400 Extra Pack is a small Java library on top of JT400 to enable easy dynamic calls for IBM i Programs or Service Programs.

Easy usage

No complex PCML file definitions or writing a lot of boilerplate code. Make calls with a few lines of code, making code looks cleaner.

Quark Engine integration.

With Quark Engine integration, exposing IBM i program calls and service program calls to the JavaScript for browser or NodeJS are easy.

Complex output formats support

Response data bytes can be easily converted into complex formats with support for deep child scan, class inheritance and field relative offsets.

What's Included

Simplification

No need for PCML or boilerplate code. Engine will use Java Dynamic Proxy technology to automatically map Java Classes to parameters or resposne formats.

Deep format scan

Data parser converter support complex definitions as deep linked child formats, arrays, relative data offsets.

Parameters annotations

Create plain Java Class, add JT400Parameter annotations to the fields, engine will convert data into IBM i program calls.

Data format annotations

Create plain Java Class, add JT400Parameter annotations to the fields, engine will convert received byte data into Java class instance.

Auto Exception

No need for JT400 API calls and checks for error messages. Program call will throw exceptions automatically including parsed error messages.

Error message output parameters

Parameter output errors are supported by IBM defined ERRC0100 and ERRC0200 formats.

How To Use It

Instead of using PCML or writing a lot of boilerplate code, imagine you can do this...
final QDCRDEVD params = QDCRDEVD.build(DEVD0100.class, "QPADEV0001");
final IQDCRDEVD program = IQDCRDEVD.create(as400);
final DEVD0100 res = program.call(params, DEVD0100.class);

System.out.println(res);
System.out.println(res.getDeviceName());
